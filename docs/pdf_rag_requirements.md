# 📋 PDF 파일 검색(RAG) 기능 요건 정의서

## 1. 프로젝트 개요
ChatX 프로젝트에 단일 PDF 파일에 대한 검색(Retrieval) 기능을 추가하여, 사용자가 PDF 내용에 대해 질문하고 관련 구간을 참조하여 답변을 받을 수 있는 시스템을 구현합니다.

## 2. 기능 요구사항

### 2.1 PDF 처리 방식
**1단계 구현 (우선순위: 높음)**
- ThreadExecutor 병렬화를 사용하여 batch 단위 처리 (페이지 수는 상수로 관리)
- 각 페이지를 base64 인코딩 후 이미지 프롬프트로 텍스트 변환
- 180초 타임아웃 제한 (하드코딩 상수로 관리)

**2단계 구현 (향후 확장)**
- Layout understanding OCR 모델 도입 (오픈소스)
- 페이지별 (직사각형좌표, 텍스트, 이미지, 메타데이터) 튜플 데이터 생성
- 외부 DB 없이 단일 파일 chunk 분할 저장

### 2.2 파일 제한사항
- 현재 50MB 제한 유지
- 배치 단위 sequential 처리 (배치 크기는 상수로 관리)
- 처리 시간이 과도할 경우 중단 메커니즘

### 2.3 PDF 지원 범위
**1단계**: 단일 PDF 파일만 지원 (수십~수백 페이지)
**2단계**: 다중 PDF 파일 지원 (파일 제목/객체로 chunk 연결)

## 3. 검색 및 RAG 기능

### 3.1 검색 방식
- 기본: 키워드 기반 검색
- 향후: Semantic search 추가 (문서 길이가 AI context 범위 내일 경우 불필요)

### 3.2 텍스트 청킹
- **구현 방식**: [Semantic chunking](https://towardsdatascience.com/how-to-chunk-text-data-a-comparative-analysis-3858c4a0997a/) 정확 재현
- 벡터 DB 및 외부 라이브러리 사용 금지
- 글자 수 기반 청킹 대신 유사도 기반 청킹

### 3.3 검색 결과 표시
- **UI 컨트롤**: Top-k 파라미터 설정용 수평 슬라이더 (1~10)
- **인용 형식**: 유사도 순서로 `[1] (Page 2) 내용...`, `[2] (Page 1) 내용...`
- **메타데이터**: 고유 ID 및 페이지 번호 정보 포함
- **하이퍼링크**: 시각적 스타일만 적용 (실제 링크 기능 제외)

## 4. 세션 관리

### 4.1 PDF 유지 방식
**1단계**: 채팅 세션 동안 메모리 유지 (DB 연결 없음)
**2단계**: 브라우저 새로고침 후에도 유지 (캐시/쿠키 활용, 구현 복잡도에 따라 후순위)

### 4.2 PDF 상태 표시
- **기존 UI**: 하단 파일 정보 표시 유지
- **추가 UI**: 채팅창 내 PDF 아이콘 + 파일명 직사각형 객체
- **향후 확장**: 미리보기 기능

### 4.3 PDF 교체 정책
- 새 PDF 업로드 시 기존 PDF 교체
- 다중 PDF 미지원 시 에러 출력

## 5. 사용자 경험 (UX)

### 5.1 로딩 상태 표시
- 배치 단위 처리 진행률 표시 로딩바
- 원형 Spinner UI 구현

### 5.2 검색 모드 전환
**우선순위 구현**:
- PDF 업로드 시 자동으로 파일 검색 모드 텍스트 출력
- 예시: "질문을 대답하기 위해 [파일명]을 조회합니다"
- 사용자 질문에 따라 텍스트 내용 변경
- 직사각형 UI를 응답 하단에 강제 표시

**향후 구현**: 
- Tool 기반 file-search 스트리밍 (구현 복잡도에 따라 후순위)

### 5.3 미리보기
- 1단계: 간단한 직사각형 UI로 대체
- 향후: 실제 미리보기 기능 추가

## 6. 기술적 구현 방식

### 6.1 데이터 저장
- **방식**: 로컬 저장 (벡터 DB 미사용)
- **형식**: Python 친화적 메타데이터 튜플
- **파일 형식**: Parquet 등 pandas 호환 형식
- **확장성**: 대용량 DB 호환 고려

### 6.2 임베딩 모델
- 1단계: Semantic chunking 구현
- 2단계: 추가 인스트럭션에 따른 임베딩 모델 결정

### 6.3 기존 코드 통합
- **백엔드**: `api/app/` 디렉토리 내 기능별 .py 파일 추가
- **프론트엔드**: 필요시 컴포넌트 파일 추가
- **기존 구조**: 현재 이미지 분석 코드와 자연스러운 통합

## 7. 성능 및 제약사항

### 7.1 응답 시간
- 최소 1개 batch 처리 필수
- 180초 타임아웃 (상수로 관리)
- 타임아웃 초과 시 자동 중단

### 7.2 리소스 관리
- **메모리**: 제약 없음
- **동시 사용자**: 고려 사항 없음 (단일 사용자 기준)

## 8. 구현 우선순위

### Phase 1 (MVP)
1. 배치 처리 시스템 (페이지 수 상수 관리)
2. 텍스트 추출 및 Semantic chunking
3. 기본 검색 기능
4. Top-k 슬라이더 UI
5. 로딩 상태 표시

### Phase 2 (확장)
1. 다중 PDF 지원
2. 브라우저 새로고침 후 유지
3. Layout understanding OCR
4. 실제 미리보기 기능

## 9. 기술 스택
- **백엔드**: FastAPI, PyMuPDF, ThreadExecutor, Pandas, Parquet
- **프론트엔드**: React, TypeScript, 기존 ChatX UI 컴포넌트
- **AI**: OpenAI API (텍스트 변환), Semantic chunking 알고리즘
